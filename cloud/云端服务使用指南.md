修改 `edge_member_a/config.json`：

```json
{
  "cloud_api_base": "http://10.61.0.137:5000",
  "rtmp_server": "10.61.0.137",
  "rtmp_port": 1935,
  "rtmp_app": "live",
  "device_id": "mobile_001",
  "device_type": "huawei_tablet",
  "scene_id": "scene_704_sandbox",
  "resolution": "1280x720",
  "fps": 15,
  "codec": "H.264",
  "bitrate": "2Mbps"
}
```

### 边端推流地址

```
rtmp://10.61.0.137:1935/live/mobile_001
```

### 边端操作步骤

1. **注册设备**：
   ```bash
   python3 edge_client.py --config config.json register
   ```

2. **开始推流**（使用平板RTMP软件或FFmpeg）：
   ```bash
   # 使用FFmpeg测试推流
   ffmpeg -re -i test.mp4 -c:v libx264 -preset veryfast -r 15 -s 1280x720 -b:v 2M -f flv rtmp://10.61.0.137:1935/live/mobile_001
   ```

3. **保持心跳**：
   ```bash
   python3 edge_client.py --config config.json watch
   ```

4. **停止后注销**：
   ```bash
   python3 edge_client.py --config config.json unregister
   ```

## 🧪 测试云端服务

### 1. 测试HTTP API

```bash
# 健康检查
curl http://10.61.0.137:5000/api/health

# 查看设备列表
curl http://10.61.0.137:5000/api/devices

# 手动注册设备

  -H "Content-Type: application/json" \
  -d '{
    "device_id": "test_001",
    "stream_url": "rtmp://10.61.0.137:1935/live/test_001",
    "resolution": "1280x720",
    "fps": 15,
    "scene_id": "test_scene"
  }'
```

### 2. 测试RTMP推流

使用FFmpeg推流测试视频：

```bash

  -c:v libx264 -preset veryfast -b:v 2M -c:a aac \
```

## 📊 云端工作流程

```
1. 边端注册设备
   ↓
2. 云端接收注册信息，记录设备
   ↓
3. 边端开始RTMP推流到 rtmp://10.61.0.137:1935/live/mobile_001
   ↓
4. 云端自动启动视频流处理线程
   ↓
5. 从RTMP拉取视频流，解帧
   ↓
6. 每3帧处理1帧（抽帧策略）
   ↓
7. 送入AI模型分析（目前为模拟数据）
   ↓
8. 通过WebSocket实时推送结果到前端
   ↓
9. 前端接收并展示分析结果
```

## ❓ 常见问题

### Q: 边端推流后云端没反应？
A: 检查设备是否已注册，查看 `curl http://10.61.0.137:5000/api/devices`

### Q: WebSocket连接失败？
A: 检查前端配置中的WebSocket地址是否正确，确认是 `ws://` 而不是 `wss://`

### Q: RTMP推流失败？
A: 确认RTMP服务器正在运行，端口1935已开放

### Q: 如何查看实时日志？
A: 查看主服务器输出或使用浏览器开发者工具查看WebSocket消息
