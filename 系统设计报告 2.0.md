# 1 引言

## 1\.1 项目背景

当前城市道路、校园交通沙盘依靠人工监控、固定老式摄像头完成管理，存在部署成本高、画面传输延迟大、缺乏智能识别能力等痛点：无法自动识别车牌、统计拥堵、追踪违停、检测路面掉落异物。

针对以上问题，本项目搭建云边端协同智慧交通视觉感知系统，采用移动端平板作为边端采集设备，独立GPU电脑作为云端AI分析服务器，PC前端作为可视化大屏终端，实现视频采集、网络传输、多模型AI推理、业务判定、实时可视化全链路闭环，适用于校园交通沙盘与小型道路场景演示。

## 1\.2 设计目标

1. 边端实现平板摄像头实时采集，以 SRT 为主低延迟推流，RTMP 作为兼容备用，网络不佳时支持短视频 HTTP 上传备用；

2. 云端完成视频解码、抽帧，集成YOLOv11车辆/车牌检测、LPRNet车牌识别、ByteTrack多目标跟踪、MOG2道路异常四大AI模型；

3. 支持四大核心业务：车牌识别白名单通行、区域拥堵热力统计、禁停违停时长告警、路面异物检测；

4. 云端通过WebSocket实时向PC前端推送结构化JSON分析结果；

5. 前端实现视频流、检测框、热力图、告警列表一体化可视化；

6. 普通校园WiFi、手机热点环境可稳定运行，满足课程演示、毕设演示实时性与稳定性要求。

## 1\.3 设计原则

1. **三端解耦原则**：边端只负责采集编码，云端承担全部AI算力，前端仅可视化，模块间通过标准接口通信，互不耦合；

2. **云端集中算力原则**：移动端算力薄弱，所有深度学习推理统一放在GPU云端，降低边端硬件门槛；

3. **实时优先原则**：采用抽帧、异步队列、旧帧丢弃策略，杜绝视频延迟持续累积；

4. **故障可降级原则**：SRT 主链路断连优先切换 RTMP 备用推流，双链路失效再本地短视频 HTTP 分段上传；

5. **可扩展原则**：设备、模型、业务场景均模块化新增，无需重构核心通信与推理框架。

## 2\.2 物理部署结构

三套独立设备支持跨网段通信，无强制同局域网约束，各设备部署与网络访问规则如下：

1. **边端设备**：华为安卓平板，预装 SRT 推流 APP；网络接入支持校园 WiFi、手机移动热点两种方式，视频流采用 SRT 协议直连云端 MediaMTX 服务，RTMP 作为兼容备用推流通道。

2. **云端服务节点**：搭载 GPU 的台式主机，统一部署 MediaMTX 流媒体服务、Flask 业务后端、全量 AI 推理模型与业务数据库；开放端口区分两类流量：

    - 媒体流端口：8890 \(UDP，SRT 主推流\)、1935 \(TCP，RTMP 备用推流\)；

    - 信令端口：5000 \(TCP，HTTP 接口、WebSocket 实时消息\)。

3. **前端访问端**：普通 PC 浏览器，通过公网地址访问云端 Web 可视化页面。

#### 跨网段互通实现方案

1. 手机热点 / 校园虚拟局域网：提供公网网络通道，平板直接与云端 8890、1935 媒体端口建立连接，**视频流量不经过 FRP 中转**；

2. FRP 内网穿透工具：仅负责后端 5000 端口 HTTP/WebSocket 信令的外网穿透，不转发 SRT/RTMP 视频大流，消除多层转发带来的延迟、丢包、断流问题。

# 4 分模块详细设计

## 4\.1 边端采集与传输模块

### 4\.1\.1 模块职责

1. 调用平板摄像头采集沙盘 / 道路连续画面；

2. 视频 H\.264 Baseline 硬件压缩，控制分辨率、帧率、码率平衡带宽与清晰度；

3. SRT 为主链路实时向云端推送视频流，RTMP 作为兼容备用推流通道；

4. 设备上电后通过 HTTP 接口注册自身 ID、流地址、拍摄场景；

5. 网络严重中断、双推流链路均失效时，本地录制短视频，通过 HTTP 分段上传兜底备份；

6. 周期性 HTTP 心跳上报设备在线状态。

## 4\.2 边端运行流程

1. 平板支架固定摄像头，对准交通沙盘；

2. 接入校园 WiFi / 手机热点 / 校园虚拟局域网，网络可直连云端媒体端口；

3. 打开 SRT 推流软件，填写云端 SRT 公网地址；

4. 设置 720P/15fps/2Mbps 编码参数，启动 SRT 推流；

5. 程序自动发送设备注册 JSON 至云端 HTTP 注册 API；

6. 后台持续心跳上报在线状态；断流弹窗提示，支持手动重连 SRT，SRT 重连失败可切换 RTMP 备用链路。

## 4\.3 视频标准参数

## 4\.4 推流地址规范（分主链路 SRT、备用 RTMP）

### 4\.4\.1 主推流 SRT（默认使用）

格式：`srt://{云端公网IP}:8890?streamid=publish:live/{device_id}`示例：`srt://106.54.10.11:8890?streamid=publish:live/tablet_001`

### 4\.4\.2 兼容备用 RTMP（老旧设备 / SRT 链路故障兜底）

格式：`rtmp://{云端IP}:1935/live/{device_id}`示例：`rtmp://192.168.10.11:1935/live/tablet_001`

device\_id：平板唯一设备编号，用于云端区分多路视频源。

json

{"device_id": "tablet_001","device_type": "huawei_tablet","srt_stream_url": "srt://106.54.10.11:8890?streamid=publish:live/tablet_001","rtmp_backup_url": "rtmp://192.168.10.11:1935/live/tablet_001","scene_id": "sandbox_704","resolution": "1280x720","fps": 15,"codec": "h264","bitrate": "2Mbps","timestamp": "2026-07-10 15:30:00","status": "online"}

## 4\.6 边端轻量预处理（无 AI 推理）

仅执行带宽优化处理，不运行任何深度学习：

1. H\.264 Baseline 硬件压缩，关闭 B 帧避免时间戳异常；

2. 限制输出帧率 15；

3. 根据网络波动动态下调分辨率、码率；

4. 帧内写入设备 ID、时间戳标识；

5. 固定视角裁剪道路有效区域，减少无效像素降低推流带宽。

## 4\.7 断流容错降级机制

1. SRT 推流断开弹窗提示设备离线；

2. 支持一键重连 SRT 主链路；

3. 网络极差时自动降低码率、分辨率保障基础推流；

4. SRT 多次重连失败，自动切换 RTMP 备用推流通道；若 RTMP 链路仍持续失败，本地截取 2 秒短视频片段，通过 HTTP multipart/form\-data 分段上传至云端备份；

5. 云端 AI 拉流侧仅缓存最新单帧图像，旧帧直接覆盖丢弃，避免推理延迟持续堆积。

## 4\.2 云端流媒体接收模块（MediaMTX）

#### 4\.2\.1 模块职责

1. MediaMTX 同时监听 8890 \(UDP\) 接收边端 SRT 主推流、1935 \(TCP\) 接收 RTMP 备用兼容流，仅接收 H\.264 Baseline 编码视频；

2. 完成 SRT/RTMP 协议解封装，提供多路协议分发出口：WHEP/WebRTC（前端低延迟播放）、RTSP/TCP（本地 AI 子进程拉流）、HLS（前端降级回退播放）；

3. 管理多路设备流会话，提供流在线状态检测、关键帧缓存，支撑新客户端快速起播；

4. 独立分流前端播放链路与 AI 推理拉流链路，两路互不阻塞，避免视频、AI 互斥卡死。

### 4\.3 视频流转分发流程

边端 SRT/RTMP 推流接入 MediaMTX → 协议解封装分流分支 1（前端播放）：流转 WHEP/WebRTC/HLS，浏览器直接渲染，不经过 AI 解码；分支 2（AI 推理）：输出 RTSP 流，由本地独立采集子进程 FFmpeg 拉流解码 → 原始帧缓存队列 → 抽帧过滤 → 送入 AI 推理线程

#### 抽帧规则（后端 AI 子进程逻辑）

原始视频 15fps，配置每 3 帧抽取 1 帧用于 AI 推理；未抽取的中间帧仅供给前端视频渲染，不执行模型推理，算力消耗降低 60%。

## 4\.3 云端AI多模型分析模块

### 4\.3\.1 车牌识别子模块

处理流程：图像帧 → YOLOv11 车辆目标检测 → YOLOv11 车牌区域检测 → 车牌裁剪（尺寸、宽高比过滤，低识别置信度提取蓝色区域二次裁剪） → LPRNet 字符推理 → CTC 去重解码 → 输出车牌号 → 白名单比对输出 JSON 示例：

json

```Plain Text
{"event_type": "plate_recognition","timestamp": "2026-07-10 15:32:10","device_id": "tablet_001","plate_number": "京A88888","is_in_whitelist": true,"decision": "allow","bbox": [110,75,215,136]}
```

# 7 核心业务完整流程

## 7\.1 实时视频双通道全链路完整流程

本系统采用**云端 SRT 上云分发、前端 WHEP 低延迟播放、本地 RTSP 独立子进程 AI 推理**双通道隔离架构，实现视频预览与 AI 视觉解耦，规避原生 FFmpeg/OpenCV 句柄卡死、画面与模型互斥问题，完整链路如下：

1. **移动端采集编码推流（沙盘拍摄端）**使用 Larix Broadcaster 手机 / 平板采集沙盘画面，仅开启视频轨，固定**720p/30fps、H\.264 Baseline、2s 关键帧间隔**，关闭 B 帧、自适应码率与音频轨消除解码冲突；以 SRT Caller 模式通过 8890 UDP 端口向云端 MediaMTX 推送流，绑定唯一流 ID `live/mobile_001`，SRT 缓冲 200ms 保障跨公网弱网抗丢包。

2. **云端 MediaMTX 流媒体服务协议转发**腾讯云 MediaMTX v1\.19\.2 接收 SRT 推流，完成多协议转封装分流：

    - 低延迟预览通道：对外暴露 WHEP/WebRTC（8889）、ICE 媒体端口 8189（TCP\+UDP）供浏览器直读；同步提供 mpegts 格式 HLS（8888）作为播放故障回退方案；

    - AI 推理专用通道：输出 RTSP over TCP 流（8554 端口），专供本地边缘 AI 服务拉流，规避 UDP 跨网丢包导致的帧破损。

3. **本地边缘隔离式视频采集（子进程级故障隔离）**本地后端启动独立`rtsp_capture_worker.py`采集子进程，通过 RTSP/TCP 拉取云端视频，由 FFmpeg 硬解码输出 BGR 原始图像帧；主进程仅维护**单帧最新缓存队列**，推理速度慢于采集时直接覆盖旧帧，杜绝帧堆积延迟；内置 8s 断流看门狗、6s 容错宽限，流停滞时自动销毁卡死子进程并重建拉流，完全隔离 FFmpeg 原生阻塞对 Web 服务的影响。

4. **帧采样调度（算力均衡控帧策略）**主服务抽帧线程按配置跳帧采样：通用视频分析每 3 帧取帧，车牌识别每 2 帧执行推理，画面叠加每帧推送；启用 FP16 半精度 CUDA 加速，平衡 GPU 负载，避免小车牌检测抢占算力。

5. **分层 AI 视觉推理管线（车辆 \+ 车牌两级检测闭环）**并行执行多模块 GPU 推理，采用分级检测架构提升沙盘小车牌识别精度：① 车辆 YOLO 检测：完成车辆定位、ByteTrack 目标跟踪，输出车辆区域掩膜与运动轨迹；② 专用车牌 YOLO 检测器：独立框选沙盘车牌（置信度阈值 0\.2），过滤过小、畸形车牌框；③ 图像预处理裁剪：收紧车牌蓝色区域边界，过滤高度＜30px、宽高比异常无效裁剪块；④ LPRNet OCR 识别：94×24 归一化输入 \+ CTC 解码，初次识别异常自动二次裁剪重试；⑤ 时序缓存稳定化：30 秒车牌 IoU 加权投票缓存，消除单帧识别抖动，同时执行车牌格式合法性校验；⑥ 交通异常建模：同步输出交通密度、违停、道路异常多维度事件。

6. **业务指标计算与结构化事件输出**基于 AI 推理原始数据完成上层业务判定：车流通行量统计、拥堵等级、违停标记、异常事件归类，统一封装标准化 JSON 事件结果，关联车辆跟踪 ID 与车牌识别文本。

7. **双链路数据推送至前端**

    - 视频画面：浏览器通过 WHEP/WebRTC 直连云端 MediaMTX 获取原始低延迟视频流，不受本地 AI 进程启停影响；

    - AI 业务数据：本地 5001 端口 Flask/Socket\.IO 长连接推送 JSON 识别结果、事件告警、车辆跟踪坐标。

8. **前端可视化渲染展示**前端画布叠加车辆 / 车牌检测框、实时交通热力图层，同步展示滚动告警列表、沙盘场景切换面板；WebRTC 流异常时自动降级切换 HLS 回放链路。

## 7\.2 车牌识别与通行判定流程

视频帧 → YOLO车牌检测 → 裁剪车牌ROI → LPRNet前向推理 → CTC去重解码车牌 → 查询白名单 → 生成放行/拦截JSON → 入库\+推送前端。

## 7\.3 车辆检测\+拥堵热力流程

视频帧 → YOLOv11全车检测 → 提取所有车辆框 → 匹配预设道路分区统计数量 → 计算拥堵等级 → 热力数据封装推送。

## 7\.4 ByteTrack违停跟踪告警流程

帧检测车辆 → ByteTrack分配全局track\_id → 判断坐标是否在禁停多边形内 → 记录首次进入时间 → 逐帧更新位置、累加停留时长 → 超过30秒阈值生成违停告警。

## 7\.5 道路异物异常检测流程

MOG2生成前景掩码 → YOLO车辆区域置黑剔除正常车辆 → 剩余前景提取连通域 → 过滤面积、连续帧数条件 → 判定路面静止异物并告警。

# 8 实时性与性能优化方案

## 8\.1 智能抽帧降负载

原始15fps，仅1/3帧执行全套AI推理，其余帧只用于前端视频渲染，GPU算力下降60%，延迟大幅降低。

## 8\.2 最新帧丢弃防阻塞

AI推理速度跟不上视频流时，队列只保留当前最新一帧，旧图像直接丢弃，不会出现延迟越积越高的现象。

## 8\.3 异步多线程流水线

拆分四大独立线程：视频解码、AI推理、业务计算、WebSocket推送，互不阻塞，单模块卡顿不影响整条链路。

## 8\.4 网络与算力降级方案

1. 网络差：自动降低码率、分辨率；

2. GPU显存不足：降低推理批次、扩大抽帧间隔；

3. SRT断流：切换RTMP上传模式。
