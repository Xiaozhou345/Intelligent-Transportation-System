## 软件概述

### 2.1 产品简介

云边端协同智慧交通视觉感知系统是一套基于"云边端协同"架构的智能交通监控系统。系统采用**三层硬件架构**：移动端平板作为边缘采集设备、GPU电脑作为云端AI分析服务器、PC作为前端监控大屏，构建视频采集-网络传输-智能分析-可视化告警完整业务闭环。

**系统部署拓扑**：

```
边端（华为平板/手机）
    ↓ SRT主推流（UDP 8890）/ RTMP备用推流（TCP 1935）
云端（GPU电脑）- MediaMTX流媒体服务
    ↓ WHEP/WebRTC（8889）播放流       ↓ RTSP（8554）AI拉流
前端（PC浏览器）                    云端AI推理引擎
    ↓ 实时视频渲染 & 可视化            ↓ WebSocket（5001）推送结果
```

**三层架构说明**：

| 层级 | 设备 | 职责 |
|------|------|------|
| 边端采集层 | 华为平板/手机 | 视频实时采集、H.264编码压缩、SRT/RTMP推流、设备注册上报 |
| 云端分析层 | GPU电脑 | MediaMTX流媒体转发、视频解码抽帧、多模型AI推理、业务逻辑判定、WebSocket推送 |
| 前端展示层 | PC浏览器 | 视频播放、检测框叠加、热力图渲染、告警列表展示、参数配置管理 |

### 2.2 核心功能清单

#### 2.2.1 车牌识别与通行决策

- YOLOv11车辆/车牌双检测，定位车牌区域
- LPRNet端到端字符识别（68类字符：31省份+数字+大小写字母）
- MySQL白名单车辆库比对，输出放行/拦截决策
- 识别事件持久化入库，支持历史查询
- WebSocket实时推送识别结果
- 车牌去重机制：30秒冷却时间防止重复推送

性能指标：车牌整体识别准确率≥95%，单帧推理耗时≤100ms。

#### 2.2.2 车辆检测与拥堵热力图

- YOLOv11s全画面车辆检测
- 自定义道路区域多边形划分
- 区域车辆计数与拥堵等级判定：
  - 畅通（绿色）：≤2辆
  - 缓行（黄色）：3~5辆
  - 拥堵（红色）：＞5辆
- ECharts热力图实时渲染，每秒刷新
- 阈值参数可动态配置

#### 2.2.3 车辆跟踪与违停告警

- ByteTrack多目标持续跟踪，分配唯一track_id
- 前端绘制多边形禁停区域，后台持久保存
- 车辆进入禁停区计时，精确到0.1秒
- 超时违停告警（阈值可配置1~300秒）
- 告警记录存储与历史查询导出
- 视频源切换时自动重置跟踪状态

#### 2.2.4 道路异常物体检测

- MOG2背景建模提取前景物体（默认）
- DINOv2固定机位参考特征检测（可选）
- YOLO车辆区域掩膜剔除正常车辆干扰
- 多时序校验过滤光影、树叶抖动误报
- 异常判定条件：
  - 连通域面积＞350像素
  - 连续4帧稳定存在无大幅位移
  - 位于可行驶路面区域
  - 不属于车辆检测框覆盖范围

#### 2.2.5 边端采集传输

- 摄像头实时采集，支持720P/1080P/480P分辨率切换
- H.264 Baseline编码，可调节码率平衡画质与带宽
- SRT主推流（UDP 8890），RTMP备用推流（TCP 1935）
- 设备注册上报与周期性心跳保活（每30秒）
- 断流备用：RTMP断流时自动截取短视频HTTP上传

#### 2.2.6 云端流媒体与调度

- MediaMTX流媒体服务，协议转封装分流
- RTSP独立子进程解码（FFmpeg+OpenCV），解耦采集与推理
- 智能抽帧：通过ITS_FRAME_SKIP环境变量配置，默认每帧处理
- 最新帧缓存：推理阻塞时丢弃旧帧，延迟不累积
- 多模型调度：YOLO、LPRNet常驻GPU，MOG2/DINOv2按需加载
- 性能监控：每30秒输出视频流性能统计

#### 2.2.7 前端可视化监控

- WHEP/WebRTC主视频播放，异常自动降级HLS
- Canvas叠加车辆框、车牌框、禁停区、异常框
- 后端绘制视频帧推送（video_frame），前端解码显示
- 四大业务分析模式切换（车牌/拥堵/违停/异物）
- 实时告警列表滚动刷新，支持分页查询历史
- 白名单增删改查可视化表单
- 事件流组件固定高度600px，支持滚动查看历史事件，新事件始终显示在最上方

#### 2.2.8 设备与系统管理

- 边缘设备管理：在线/离线状态、推流参数查看、视频流处理状态
- 模型配置管理：推理置信度、NMS阈值远程配置（仅管理员）
- 告警阈值管理：违停超时、拥堵分级阈值、置信度可视化配置（仅管理员）
- 性能监控：CPU、GPU显存、内存使用率定时采集展示
- 权限管控：区分系统管理员/普通用户操作权限

#### 2.2.9 数据统计与历史查询

- 车牌通行统计：按时间段统计进出车辆数量
- 拥堵趋势统计：各区域拥堵时长日/小时折线图
- 违停告警统计：每日违停次数、高发区域统计
- 道路异常统计：异物告警频次、车道分布
- 多条件查询：按设备、时间、事件类型筛选
- 数据导出：统计历史、告警记录导出

#### 2.2.10 参数配置面板

系统配置面板（仅管理员可见）分为三个标签页：

| 标签页 | 配置项 | 说明 |
|--------|--------|------|
| AI推理参数 | 车辆检测置信度、车牌检测置信度 | 调整YOLO检测阈值 |
| 跟踪算法参数 | 违停判定停留时长 | 设置违停告警时间阈值（1~300秒） |
| 业务阈值参数 | 畅通阈值、缓行阈值、拥堵阈值 | 配置拥堵热力图分级标准 |

配置面板支持：
- 从后端同步运行时参数
- 实时下发修改后的参数到后端
- 恢复默认配置功能

### 2.3 技术特性

- **三端解耦原则**：采集、算力、展示三层完全解耦，通过标准接口通信
- **多模型动态调度**：高频YOLO、LPRNet常驻GPU，低频MOG2/DINOv2按需加载释放显存
- **实时性优化**：智能抽帧、异步多线程流水线、旧帧丢弃机制、缓冲区高频清理（每0.5秒）
- **双传输容错机制**：SRT主推流（UDP 8890）为主、RTMP（TCP 1935）为兼容备用，断流自动切换
- **无监督异常检测**：背景建模+车辆掩膜识别道路未知掉落物，支持MOG2和DINOv2双后端
- **FRP穿透隔离**：仅5001端口HTTP/WebSocket信令使用FRP穿透，视频大流直连
- **视频流性能监控**：每30秒输出读取帧率、处理帧率、缓冲清理耗时等统计数据
- **角色权限控制**：管理员可注册设备、配置参数、管理白名单；普通用户仅可查看和处置告警
- **帧序列检查**：前端解码时检查帧序号，丢弃旧帧避免延迟累积
- **用户注册功能**：支持新用户注册，可选择管理员或普通用户角色

## 安装与初始配置

### 4.1 获取安装包

系统源码托管于 GitHub，可通过以下方式获取：

```bash
git clone https://github.com/Xiaozhou345/Intelligent-Transportation-System.git
cd Intelligent-Transportation-System
```

### 4.2 安装步骤

#### 4.2.1 云端服务器部署

1. **安装 MediaMTX**

```bash
cd ~/mediamtx
# 上传 deploy/mediamtx/cloud-mediamtx.yml 配置文件
./mediamtx ./cloud-mediamtx.yml
```

MediaMTX配置说明：
- 监听协议：RTSP（8554）、RTMP（1935）、SRT（8890）、HLS（8888）、WebRTC（8889）
- 协议转封装：SRT/RTMP输入 → WHEP/WebRTC、RTSP、HLS输出

2. **安装 frp（可选，用于内网穿透）**

```bash
cd ~/frp_0.69.1_linux_amd64
./frps -c frps.toml
```

注意：FRP仅穿透5001端口HTTP/WebSocket信令，不转发SRT/RTMP视频大流。

#### 4.2.2 本地AI电脑部署

1. **安装 Python 依赖**

```bash
cd cloud
pip install -r requirements.txt
```

2. **配置数据库**

创建 MySQL 数据库：
```sql
CREATE DATABASE intelligent_transportation_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

配置数据库连接（创建 `.env.db` 文件）：
```bash
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=intelligent_transportation_system
```

3. **下载模型权重**

- YOLOv11s：`yolo11s.pt`（约19MB）
- 沙盘车辆模型：`sandbox_vehicle_best.pt`
- 沙盘车牌模型：`sandbox_plate_best.pt`
- LPRNet：`sandbox_lprnet_best.pth`（约1.8MB）
- 道路分割模型：`sandbox_drivable_best.pt`

#### 4.2.3 前端部署

```bash
cd ui
npm install
npm run build
```

### 4.3 初始配置与激活

#### 4.3.1 环境变量配置

**前端配置（`ui/.env`）**：
```bash
VITE_CLOUD_SERVER_URL=http://106.54.10.11:15000
VITE_LOCAL_API_PORT=5001
VITE_LIVE_VIDEO_URL=http://106.54.10.11:8888/live/mobile_001/index.m3u8
VITE_LIVE_WEBRTC_URL=http://106.54.10.11:8889/live/mobile_001/whep
VITE_SIMULATION_MODE=false
```

**边端配置（`edge_member_a/config.json`）**：
```json
{
  "device_id": "mobile_001",
  "device_type": "huawei_tablet",
  "stream_url": "rtmp://<server_ip>:1935/live/mobile_001",
  "scene_id": "scene_704_sandbox",
  "api_base_url": "http://<server_ip>:5001/api"
}
```

**后端环境变量**：
```bash
ITS_PORT=5001
ITS_FRAME_SKIP=1
ITS_VEHICLE_CONF=0.60
ITS_PLATE_CONF=0.20
ITS_PLATE_COOLDOWN=30.0
ITS_VEHICLE_COOLDOWN=30.0
ITS_ANOMALY_BACKEND=dino_reference
ITS_ENABLE_SANDBOX_AI=true
ITS_ASYNC_MODEL_INIT=true
```

#### 4.3.2 设备注册

```bash
python edge_member_a/edge_client.py --config edge_member_a/config.json register
```

或通过 HTTP API 注册：
```bash
curl -X POST http://localhost:5001/api/register_device \
  -H "Content-Type: application/json" \
  -d '{
    "device_id": "tablet_001",
    "device_type": "huawei_tablet",
    "stream_url": "rtsp://<mediamtx_ip>:8554/live/tablet_001",
    "scene_id": "sandbox_704",
    "resolution": "1280x720",
    "fps": 15,
    "codec": "h264",
    "bitrate": "2Mbps"
  }'
```

#### 4.3.3 启动服务

**云端服务器**：
```bash
# 启动 MediaMTX
./mediamtx ./cloud-mediamtx.yml

# 启动 frps（可选）
./frps -c frps.toml
```

**本地AI电脑**：
```bash
# 启动后端API服务
cd cloud/stream_receiver
python main_server.py

# 启动前端开发服务
cd ui
npm run dev
```

### 4.4 卸载步骤

1. **停止服务**

```bash
# 停止后端服务
Ctrl+C 终止 main_server.py

# 停止前端服务
Ctrl+C 终止 npm run dev

# 停止云端 MediaMTX
Ctrl+C 终止 mediamtx
```

2. **删除项目文件**

```bash
rm -rf Intelligent-Transportation-System
```

## 核心业务流程

### 6.1 实时视频双通道全链路流程

```
1. 移动端采集编码推流（Larix Broadcaster）
   - 720p/30fps、H.264 Baseline、2s关键帧间隔
   - SRT Caller模式推流至云端MediaMTX 8890端口

2. 云端MediaMTX协议转发
   - WHEP/WebRTC（8889）→ 前端低延迟播放
   - HLS（8888）→ 前端降级回退
   - RTSP（8554）→ 本地AI子进程拉流

3. 本地边缘隔离式视频采集（独立子进程）
   - FFmpeg硬解码输出BGR原始图像帧
   - 单帧最新缓存队列，推理慢时覆盖旧帧
   - 缓冲区高频清理（每0.5秒），降低延迟

4. 帧采样调度
   - 通用分析：按ITS_FRAME_SKIP配置抽帧（默认每帧）
   - 车牌识别：每2帧执行推理
   - 画面叠加：每帧推送

5. 分层AI视觉推理管线
   - 车辆YOLO检测 → ByteTrack跟踪
   - 专用车牌YOLO检测器（置信度0.2）
   - 图像预处理裁剪（94×24归一化）
   - LPRNet OCR识别 + CTC解码
   - 30秒车牌IoU加权投票缓存

6. 业务指标计算与事件输出
   - 车流通行量统计、拥堵等级、违停标记、异常事件归类
   - 统一封装标准化JSON事件结果

7. 双链路数据推送至前端
   - 视频画面：WHEP/WebRTC直连MediaMTX
   - AI业务数据：Flask/Socket.IO 5001端口推送
   - 后端绘制完整帧（video_frame）：Base64编码推送

8. 前端可视化渲染展示
   - Canvas叠加检测框、热力图层
   - 滚动告警列表、场景切换面板
   - WebRTC异常自动降级HLS
   - 视频帧序列检查，丢弃旧帧避免延迟累积
```

### 6.2 车牌识别与通行判定流程

```
视频帧 → YOLO车辆检测 → YOLO车牌区域检测 → 车牌裁剪 → 
LPRNet字符推理 → CTC去重解码 → 白名单比对 → 
生成放行/拦截JSON → 入库 + WebSocket推送前端（30秒去重）
```

### 6.3 车辆检测与拥堵热力流程

```
视频帧 → YOLOv11全车检测 → 提取车辆坐标 → 
匹配预设道路分区统计数量 → 计算拥堵等级（畅通/缓行/拥堵）→ 
热力数据封装推送前端ECharts渲染
```

### 6.4 ByteTrack违停跟踪告警流程

```
YOLO车辆检测 → ByteTrack分配track_id → 
判断坐标是否在禁停多边形内 → 记录首次进入时间 → 
逐帧更新位置、累加停留时长 → 超过阈值生成违停告警 → 
入库 + WebSocket推送前端 → 视频源切换时重置跟踪状态
```

### 6.5 道路异物异常检测流程

```
视频帧 → MOG2/DINOv2背景建模提取前景 → 
YOLO车辆区域掩膜剔除车辆 → 剩余连通域筛选 → 
面积>350像素且连续4帧稳定 → 判定路面异物 → 
生成异常告警 → 入库 + WebSocket推送前端
```

### 6.6 参数配置同步流程

```
前端打开配置面板 → 请求后端配置（/api/config）→ 
后端返回运行时参数 → 前端渲染配置项 → 
用户修改参数 → 点击保存配置 → 
WebSocket发送update_config指令 → 
后端更新配置 → 前端同步默认值
```

### 6.7 用户注册与登录流程

```
用户注册：
输入用户名 → 输入密码 → 确认密码 → 选择角色 → 提交注册 → 
后端验证用户名唯一性 → 密码哈希存储 → 创建用户记录 → 自动登录

用户登录：
输入用户名 → 输入密码 → 选择角色 → 提交登录 → 
后端验证用户存在 → 验证密码哈希 → 更新最后登录时间 → 
返回用户信息 → 前端存储用户状态 → 展示主界面
```

## 常见问题与异常处理

### 9.1 常见报错及解决

#### 9.1.1 视频画面无法加载

**问题现象**：前端界面显示"未连接"状态，视频区域为空白

**可能原因**：
- 边端设备未启动推流
- 云端 MediaMTX 服务未启动
- 网络连接异常
- 端口未正确放行

**解决方法**：
1. 检查边端设备是否正在推流
2. 确认云端 MediaMTX 服务状态：`curl http://<server_ip>:8888/`
3. 检查网络连通性：`ping <server_ip>`
4. 确认安全组/防火墙已放行相关端口（8888、8889、5001）

#### 9.1.2 检测结果延迟过高

**问题现象**：视频画面正常，但检测框显示延迟明显

**可能原因**：
- 网络延迟过高
- AI推理速度慢
- 跳帧策略过于保守

**解决方法**：
1. 检查网络延迟：观察前端延迟显示（正常 < 200ms）
2. 启用 CUDA 加速：确保GPU可用
3. 调整跳帧策略：增加ITS_FRAME_SKIP环境变量值
4. 检查缓冲区清理：后端每30秒输出性能统计

#### 9.1.3 违停告警误报

**问题现象**：正常行驶车辆被误判为违停

**可能原因**：
- 禁停区域设置过大
- 停留时长阈值设置过小
- 目标跟踪丢失

**解决方法**：
1. 调整禁停区域范围
2. 增加停留时长阈值（建议 ≥ 30秒）
3. 检查 ByteTrack 跟踪参数

#### 9.1.4 道路异常检测不灵敏

**问题现象**：路面异常物体未被检测到

**可能原因**：
- 背景模型未正确初始化
- 光照变化影响检测
- 异常物体面积过小

**解决方法**：
1. 重新初始化背景模型（确保道路为空时初始化）
2. 调整异常检测阈值
3. 确保异常物体面积大于最小检测阈值（>350像素）
4. 尝试切换异常检测后端（MOG2/DINOv2）

#### 9.1.5 SRT推流频繁断开

**问题现象**：边端设备推流不稳定，频繁断开

**可能原因**：
- 网络信号差
- SRT缓冲设置不合理
- 校园网端口隔离

**解决方法**：
1. 切换至手机热点直连云端
2. 增加SRT缓冲时间（建议200ms）
3. 检查校园网是否封锁8890端口

#### 9.1.6 配置面板打开后为空白

**问题现象**：管理员点击设置图标后，配置面板打开但内容为空

**可能原因**：
- 后端配置接口未正常响应
- WebSocket连接异常

**解决方法**：
1. 检查后端服务是否正常运行
2. 检查浏览器控制台是否有错误信息
3. 刷新页面后重新登录

#### 9.1.7 普通用户看不到设置按钮

**问题现象**：普通用户登录后右上角没有设置图标

**原因说明**：
- 设置按钮仅管理员可见，普通用户无权限配置参数

**解决方法**：
- 使用管理员账号登录

#### 9.1.8 用户注册失败

**问题现象**：点击注册按钮后提示注册失败

**可能原因**：
- 用户名已存在
- 密码和确认密码不一致
- 数据库连接异常

**解决方法**：
1. 尝试使用其他用户名
2. 确保密码和确认密码一致
3. 检查后端数据库连接状态

### 9.2 数据备份与恢复

#### 9.2.1 备份

系统数据存储于 MySQL 数据库，定期备份建议：

   ```bash

   ```

   ```bash

   ```

## 风险识别与应对措施

| 风险 | 应对方案 |
|------|----------|
| 校园网端口隔离，平板无法连云端 | 使用手机热点直连或FRP内网穿透 |
| LPRNet训练识别准确率低、乱码 | 扩充沙盘车牌数据集、调整学习率、修复CTC空白冲突 |
| GPU显存不足多模型冲突 | 推理串行执行，降低输入分辨率，异步模型初始化 |
| 视频延迟持续累积 | 启用最新帧丢弃、增大抽帧间隔、高频缓冲区清理 |
| SRT推流频繁断开 | 增加心跳重连，备用分段上传方案 |
| 违停跟踪ID跳变 | 调优ByteTrack卡尔曼滤波匹配阈值，视频源切换时重置跟踪状态 |
| 异常检测光影误报 | 增加光影过滤算法，支持MOG2/DINOv2双后端切换 |
| 普通用户误操作 | 通过角色权限控制，限制普通用户的配置权限 |
| 用户注册安全风险 | 使用bcrypt密码哈希存储，验证用户名唯一性 |
